## 1c. 📧 KNOWN ISSUE — signup fails until email sender is fixed

**Symptom:** creating an account returns `500: Error sending confirmation email`
(`POST /auth/v1/signup` → 500). Not a code/DB bug — the app calls a standard
`supabase.auth.signUp()`.

**Root cause (from Auth logs):**
`gomail: could not send email: 550 "The yourdomain.com domain is not verified"`.
The Supabase custom SMTP is wired to **Resend**, but the **Sender ("From") email is still
the placeholder `…@yourdomain.com`**, which is not a verified Resend domain — so every
confirmation email is rejected and the signup 500s. Resend also refuses free/public domains
(e.g. `gmail.com`) — it only accepts a domain you own.

**Temporary fix (applied for testing):** Supabase → **Authentication → Providers → Email →
turn OFF "Confirm email"**. Signups then succeed for any email with no email sent.
⚠️ This is a **testing-only** state — it must be re-enabled before real users.

**Permanent fix (do before launch — still a P0 email item):**
1. Register a domain you own (~$1–12/yr; Cloudflare / Porkbun / Namecheap). Brand fit:
   `carbonify.com` or a variant — the app already uses `support@carbonify.com`.
2. **Resend → Domains → Add Domain** → add the SPF/DKIM DNS records at the registrar →
   wait for **Verified**.
3. **Supabase → Authentication → Emails → SMTP Settings**: Sender email = an address on that
   domain (e.g. `no-reply@carbonify.com`), host `smtp.resend.com`, port `465`/`587`,
   user `resend`, password = Resend API key.
4. Re-enable **"Confirm email"** and re-test signup.

> Note: `onboarding@resend.dev` works without a domain but only delivers to your own Resend
> account email — not usable for real signups. Owning a domain is required.

## 3. 🧪 Test today (after applying migration 19)

**Project Developer**
1. Log in as a developer → you should land on **My Project Submissions**.
2. With no projects, you see the **"Submit your first project"** hero.
3. Submit a project. Try submitting with missing docs → blocked with a clear list.
4. Attach all required PDFs + submit → succeeds.
5. Open the project detail (or have a verifier open it) → **each document link opens the PDF**.

**Verifier**
1. Log in as a verifier → lands on the Verifier panel; queue rows show **coloured status badges**.
2. Open a submitted project → fill the rubric; try **Validate** with the rubric incomplete → you get a warning.
3. Set a price + Validate → project validates, price persists (no false success).
4. MRV review: try **Reject** with no note → blocked; with a note → confirms then rejects.

**LGU**
1. Log in as an LGU user → lands on **LGU Tools**.
2. MSW calculator: enter diverted > generated → rejected with a message.
3. Save a valid record → appears in Records; diversion rate ≤ 100%.
4. Endorsements: open "Review project details", add a note, Endorse/Decline.
